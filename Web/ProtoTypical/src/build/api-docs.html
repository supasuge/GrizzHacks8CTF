Proto-Palooza

“Internal user settings, preferences, notifications, and profile management — all behind one neat little API.”

Live v1.2.0 Sessionless Demo
Base URL http://localhost:3000
Endpoints
GET / Service index ▶

Returns general service metadata and a map of available routes.

Response
200 OK 
{
  "message": "Welcome to Proto-Palooza!",
  "version": "1.2.0",
  "service": "user-settings-api"
}
GET /health Health check ▶

Basic liveness probe for orchestration and container monitoring.

Response
200 OK 
{
  "ok": true,
  "service": "proto-palooza",
  "version": "1.2.0"
}
GET /user Current profile record ▶

Returns the user object associated with the current client session model.

Response
200 OK 
{
  "user": {
    "id": "usr_x8d2n4zf",
    "username": "guest_x7k2m9",
    "bio": "Just a regular user.",
    "theme": "dark",
    "preferences": { ... }
  }
}
GET /user/preferences Preference object ▶

Fetches the nested user preference object used by the dashboard and editor surfaces.

Response
200 OK 
{
  "preferences": {
    "editorMode": "vim",
    "dashboardLayout": "compact",
    "telemetryLevel": "minimal"
  }
}
GET /user/activity Recent actions ▶

Returns the latest server-side activity events stored for the user object.

Response
200 OK 
{
  "recentActivity": [
    {
      "action": "account_created",
      "ts": "2026-03-19T00:00:00.000Z"
    }
  ]
}
POST /update Merge profile data ▶

Accepts a JSON object and merges it into the current profile record.

Example request
{
  "username": "hacker",
  "bio": "I break things for fun.",
  "preferences": {
    "dashboardLayout": "expanded"
  }
}
Body schema
FieldTypeDescription
usernamestringDisplay name
biostringProfile biography
themestringUI mode
preferencesobjectNested preference values
Response
200 OK 
{
  "message": "Profile updated!",
  "requestId": "req_h3af92qz",
  "user": { ... }
}
POST /user/preferences Update preferences ▶

Updates the nested preferences object used by the application shell.

Example request
{
  "editorMode": "nano",
  "telemetryLevel": "verbose"
}
Response
200 OK 
{
  "message": "Preferences updated!",
  "preferences": { ... }
}
GET /notifications Notification settings ▶

Fetches current delivery settings for system notifications.

Response
200 OK 
{
  "notificationSettings": {
    "email": false,
    "push": false,
    "securityAlerts": true
  }
}
POST /notifications/test Queue simulated test notification ▶

Queues a fake notification event for dashboard testing.

Example request
{
  "channel": "security"
}
Response
200 OK 
{
  "queued": true,
  "channel": "security",
  "etaSeconds": 3
}
POST /apikeys Create API key metadata ▶

Creates a mock API key record and returns the generated token once.

Response
201 Created 
{
  "message": "API key created.",
  "apiKey": "ppz_...",
  "metadata": { ... }
}
